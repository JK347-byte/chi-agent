## Step 1：搜尋醫學文獻

針對使用者提供的主題，用 WebSearch 搜尋：

- 搜尋關鍵詞範本：`[主題] clinical guideline 2023 2024`、`[主題] systematic review`、`[主題] 台灣耳鼻喉科`
- 目標：找到具體數據（發生率、治癒率、研究樣本數）、最新臨床建議、或常見誤解的反例
- 把找到的 2–3 個有用事實整理成筆記，後續寫文章時自然帶入

## Step 2：撰寫衛教文章

### 格式要求

- 語言：繁體中文
- 長度：600–900 字
- 結構：不要公式化標題，用自然段落推進
- 視角：詹士奇醫師（以「我」或「在診間」帶入臨床觀察）

### Humanizer-zh-TW 去 AI 化規則（24 條）

寫作時主動過濾以下模式：

**內容層面**
- 不過度渲染「重要性」或「宏觀意義」
- 不堆疊「研究顯示」、「有證據表明」等模糊歸因
- 結尾不用「展望未來」、「挑戰與機遇」公式
- 不用廣告式語句（「全面守護您的健康」）

**語言層面**
- 禁用高頻 AI 詞：此外、至關重要、深入探討、格局、賦能、彰顯、豐富、探索、寶貴、充滿活力、不可或缺
- 不做否定式排比（「不是 A，而是 B；不是 C，而是 D」）
- 不過度用「三段式」結構（第一、第二、第三）
- 不刻意換同義詞（「症狀」就說「症狀」，不要輪流換成「表現」、「徵兆」）

**風格層面**
- 不用破折號（——）連接句子
- 不用粗體強調日常詞彙
- 不用 Emoji
- 不用「」以外的引號格式

**語氣層面**
- 不奉承讀者（「感謝您如此用心」）
- 不加免責聲明（「本文僅供參考，請諮詢醫師」這類結尾）
- 不過度限定條件（「在某些情況下，有可能…」）
- 結尾要有具體收尾，不寫通用正面結論

### 核心人性化原則

真實感來自這幾件事：
- **有立場**：對事實做出反應，不只是陳述。例如：「這個數字每次看都讓我覺得有點沉重。」
- **節奏不一致**：刻意讓長句和短句交替出現。
- **承認複雜**：「這件事沒有那麼簡單」比「研究證實最佳解法是 X」更真實。
- **適度用第一人稱**：「我在門診常遇到」比「臨床上常見」更有溫度。
- **感官細節**：「鼻腔深處像是積了一層薄膜」比「鼻塞症狀」更具體。

## Step 3：更新封面 HTML

模板路徑：`/Users/janskey/Downloads/Chi-Agent/cover_demo.html`
圖片資產：`/Users/janskey/Downloads/Chi-Agent/assets/photos/`

更新 HTML 中的四個欄位（直接編輯文字內容，不改 CSS）：

| HTML 元素 | 填入內容 |
|-----------|----------|
| `.category-tag` | 文章分類（例：耳鼻喉健康知識） |
| `.article-title` | 文章標題，保留 `<br>` 換行，在自然斷句處分行 |
| `.article-subtitle` | 核心主張一句話，簡短有力 |
| `.key-points` 的三個 `<span>` | 三個重點，各一行，20 字以內 |

### 形象照選擇邏輯

依文章情境選照片，不一定都用預設的 `jk_stand_smile.jpg`：

| 情境 | 推薦照片 |
|------|----------|
| 輕鬆衛教、日常健康知識 | `jk_stand_smile.jpg` |
| 嚴肅主題（手術、慢性病） | `jk_sit_confident.jpg` |
| 重點說明、導讀型 | `jk_stand_present.jpg` |
| 診所介紹、歡迎性質 | `jk_stand_welcome.jpg` |

## Step 4：截圖輸出

1. 確認本地伺服器是否在執行（port 8765）：
   ```bash
   lsof -i :8765 | grep LISTEN
   ```
   若無，啟動：
   ```bash
   cd /Users/janskey/Downloads/Chi-Agent && python3 -m http.server 8765 &
   ```

2. 用 Playwright 導航並截圖：
   - 視窗大小：1920 × 1080
   - 網址：`http://localhost:8765/cover_demo.html`
   - 輸出路徑：`/Users/janskey/Downloads/Chi-Agent/cover_output.jpeg`

3. 截圖完成後，讀取圖片確認畫面正確，再告知使用者完成。
